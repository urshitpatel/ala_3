# 📱 Flutter Local Storage App

A Flutter mobile application demonstrating local data storage using SQLite, Hive, or SharedPreferences. This project showcases how to store, retrieve, and update data efficiently without requiring an internet connection. It is developed as part of an Assignment Learning Activity (ALA).

---

## 📌 Project Overview

The **Flutter Local Storage App** is designed to help students understand persistent data storage in mobile applications. The app provides a simple and user-friendly interface that allows users to manage data using CRUD operations (Create, Read, Update, and Delete). All data is stored locally on the device, ensuring fast performance and offline accessibility.

---

## 🎯 Objectives

- Develop a cross-platform mobile application using Flutter.
- Implement local data storage using SQLite, Hive, or SharedPreferences.
- Demonstrate storing, retrieving, and updating data.
- Design a clean and intuitive user interface.
- Upload the project to GitHub with proper documentation.

## ⚙️ Installation and Setup

Follow these steps to run the project on your local machine.

### 1️⃣ Prerequisites

Ensure the following are installed:

- Flutter SDK
- Dart SDK
- Android Studio or Visual Studio Code
- Git
- Android Emulator or Physical Device

Check Flutter installation:

```bash
flutter doctor

2️⃣ Clone the Repository

git clone https://github.com/your-username/flutter-local-storage-app.git
cd flutter-local-storage-app


3️⃣ Install Dependencies
flutter pub get


4️⃣ Run the Application
flutter run


📦 Dependencies

Add the required dependencies in the pubspec.yaml file based on your chosen storage method.

🔹 SQLite
dependencies:
  flutter:
    sdk: flutter
  sqflite: ^2.3.0
  path: ^1.9.0
🔹 Hive
dependencies:
  flutter:
    sdk: flutter
  hive: ^2.2.3
  hive_flutter: ^1.1.0
🔹 SharedPreferences
dependencies:
  flutter:
    sdk: flutter
  shared_preferences: ^2.2.2

✅ Use only one storage method as per project requirements.

🚀 How It Works

The user enters data through the input form.
The data is stored locally using SQLite, Hive, or SharedPreferences.
Stored records are displayed on the home screen.
Users can update or delete existing entries.
All changes are saved and persist even after restarting the application.
